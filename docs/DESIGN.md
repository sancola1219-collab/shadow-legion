# 我的方塊世界2：陰影軍團 — 設計文件（2026-07-11）

## 靈感來源

Roblox 遊戲《Raise Your Shadow Army》（Studio Cochi）之玩法「致敬」：
進入閘口 → 打倒敵人 → 收集靈魂 → 奪取敵人影子加入不死軍團 → 回聖所讓軍團掛機產靈魂 →
升級 → 打更深區域與魔王 → 解鎖新世界 → 重生拿倍率。
**所有程式、美術、名稱皆為原創**，僅玩法類型與畫面構成參考截圖。

## 技術基礎

沿用《我的方塊世界》自製引擎（純前端、零依賴、手寫 WebGL2、手機電腦皆可玩）：

- 原封重用：`noise.js`、`world.js`、`mesher.js`、`physics.js`、`weather.js`、`input.js`
- 小改重用：`render.js`（新生物模型、第三人稱）、`audio.js`（新音效）
- 全新撰寫：`blocks.js`（本作方塊表）、`worldgen.js`（場景式地圖）、`units.js`（單位與戰鬥 AI）、
  `gamedata.js`（世界/經濟/升級/稱號資料）、`army.js`（軍團收藏/產出/奪取）、`save.js`、
  `textures.js`（本作材質）、`main.js`、`index.html`、`css`

## 視角與操作

- **第三人稱**：鏡頭在玩家後上方（可調距離），玩家為金甲勇者模型。
- 攻擊為「揮擊」：命中面前 2.6 格內扇形的敵人（不需精準瞄準，適合手機）。
- **自動模式**：開啟後自動走向最近敵人並攻擊（掛機）。按鈕同截圖「自動開啟/關閉」。
- 手機：左搖桿移動、右半屏滑動視角、攻擊/召喚等觸控按鈕。

## 場景（seed 即場景 id）

- `seed 2000`＝**聖所**：石板廣場、8 座召喚平台（部署的影子站上面顯示 靈魂/s）、
  中央閘口傳送門、商店/索引檯（走近提示，UI 按鈕也可開）。
- `seed 1000+w`＝**戰鬥世界 w（1..5）**：沿 +x 的線性道路，
  三個區域（Zone 1/3、2/3、3/3）＋終點魔王競技場；殺滿數量開門（拆除閘門方塊）。
  - W1 陰影墓園：紫天、橘樹、墓碑（對應截圖）
  - W2 冰霜墓地：雪地、冰藍
  - W3 熔岩要塞：黑石、熔岩橘
  - W4 毒霧沼澤：綠霧、沼澤
  - W5 虛空王座：深紫黑、星空

## 單位（5 世界 × 5 種 = 25 種，索引圖鑑用）

每世界：骨架系（常見）、殭屍系（常見）、守衛A（稀有）、守衛B（史詩）、魔王（傳說）。
W1 命名：骨架、殭屍、毒藥守衛、閃電守衛、骸骨領主。其餘世界依主題改名。
守衛/魔王會發射投射物。數值隨世界指數成長。

## 經濟

- **靈魂**：擊殺掉「紫焰」飄向玩家；HUD 左下顯示總數（K/M 縮寫）。
- **奪取（Possess）**：擊殺時擲骰——常見 30%、稀有 15%、史詩 6%、傳說（魔王）首殺必得。
  奪取率可升級。奪得單位入軍團收藏（同種可疊，計數）。
- **軍團**：
  - 出戰：最多 N 隻（升級 3→8）跟隨玩家自動戰鬥。
  - 聖所平台：自動選最強 8 隻展示，全收藏皆貢獻 靈魂/s（依稀有度×世界）。
  - 離線收益：關閉期間以 50% 效率累積（上限 8 小時），開啟時發放。
- **商店升級**：攻擊、生命、奪取率、軍團人數、靈魂加成（費用指數成長）。
- **重生**：打敗 W5 魔王解鎖；重置靈魂與商店升級，保留圖鑑，
  每次重生 +100% 靈魂獲得、+25% 攻擊，重生次數顯示。

## UI（繁中，配色仿截圖紫黑金）

頂部區域進度條（骷髏節點）、左下靈魂數、左側背包(軍團)按鈕、右側商店/索引/稱號、
戰鬥中：掉落數、返回、自動開關、Possess 提示；敵人頭上 HTML 投影名牌＋血條。
聖所：NEXT WEATHER 倒數（沿用天氣系統）、召喚平台名牌。

## 鐵律（沿用前作）

1. 遊戲狀態只在 60Hz `tick()` 內變；rAF 只渲染。
2. 邏輯層檔案不碰 DOM（`typeof module` 雙載入、IIFE 包裹防撞名）。
3. 隨機一律 mulberry32/座標雜湊，決定性可測。
4. 觸控＝合成輸入，不另開邏輯分支。
5. `node --test` 全綠才 commit。

## 存檔（localStorage `shadowlegion.save.v1`）

souls、upgrades、collection（各單位數量）、deploy 設定、worldProgress（最高世界/首殺記錄）、
rebirths、titles、lastSeen（離線收益）、settings。
encodeSave 為欄位白名單；版本欄 v 與 KNOWN_VERSIONS 同前作規則。
